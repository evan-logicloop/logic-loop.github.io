LogicLoop lets you run your rules on a schedule and set up automated actions. No need to spend time manually refreshing dashboards or exporting data.

You can use actions to raise alerts to monitor your processes, or integrate your rules into downstream workflows like Zapier or even custom internal flows. You can set up actions after every scheduled rule.

Note: Actions don’t work on rules that have parameters.

You can add an action to your rule by clicking the ‘+ Add Action’ button in the Rule source view.



Now you can configure your new action

  • • Name
  • • Trigger when Add a condition for your action to be triggered. Currently, the condition is based on the first row. Set the condition to an impossible value to always run the action.
  • • When triggered, send Configure how often you want the Action to be triggered. We support ‘Just once’, ‘Each time action is evaluated’, ‘At most every…’
  • • Template Customize what data is passed into the action, or use our default. See the full list of action parameters here.




Once you click ‘Create Action’, you’ll be able to add your existing Action Destinations to the action. Learn how to set up Action Destinations here.



Click ‘+ Add’ to add any action destinations configured in the Destinations tab. As the rule author, you can opt to be notified too.



You can now click ‘Run Rule Now’ to test your actions. The status of your action on the right bar will change from ‘Unknown’ to ‘Triggered’ or ‘Not Triggered’. You’ll want to ensure your rule’s query runs correctly before triggering the action.
